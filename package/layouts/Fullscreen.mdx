# FullscreenLayout

An edge-to-edge layout for immersive experiences like presentations, media players, or chat interfaces.

## Semantic Structure

```html
<body>
  <header>
    <Controls />
  </header>

  <main>
    <Slot />
  </main>

  <footer>
    <Progress />
  </footer>
</body>
```

## CSS

```css
body {
  display: flex;
  flex-direction: column;
  height: 100vh;
  overflow: hidden;
}

body > header {
  position: absolute;
  top: 0;
  left: 0;
  right: 0;
  z-index: 50;
  padding: var(--spacing-4);
  background: linear-gradient(to bottom, rgba(0,0,0,0.5), transparent);
  opacity: 0;
  transition: opacity 0.3s;
}

body:hover > header,
body:focus-within > header {
  opacity: 1;
}

body > main {
  flex: 1;
  display: flex;
  overflow: hidden;
}

body > footer {
  position: absolute;
  bottom: 0;
  left: 0;
  right: 0;
  z-index: 50;
  padding: var(--spacing-2) var(--spacing-4);
}
```

## Variants

### Slides
```css
body > main {
  display: flex;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
}
body > main > section {
  flex: 0 0 100vw;
  scroll-snap-align: start;
}
```

### Chat
```css
body > main {
  flex-direction: column;
}
body > main > section {
  flex: 1;
  overflow-y: auto;
}
body > footer {
  position: relative;
  border-top: 1px solid var(--border);
}
```

### Media
```css
body > main > video,
body > main > img {
  width: 100%;
  height: 100%;
  object-fit: contain;
}
```

## Slots

| Slot | Element | Required |
|------|---------|----------|
| Controls | `header` | No |
| Slot | `main` | Yes |
| Progress | `footer` | No |

## Mobile (React Native)

```tsx
<View style={{ flex: 1 }}>
  <StatusBar hidden />
  <View style={StyleSheet.absoluteFill}>
    {children}
  </View>
  {showControls && <Controls />}
</View>
```
